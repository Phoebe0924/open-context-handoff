# OCH V0 Audit Notes

## Improved

- Added a clear README → quickstart → prompts → examples → test-record path.
- Made the manual paste sequence, human review, and success criteria explicit.
- Added stricter section purposes, length limits, omissions, and common mistakes.
- Shortened prompts around recoverable anchors rather than conversation summaries.
- Expanded examples and test records to expose failure instead of assuming success.

## Still out of scope

V0 remains a manual, documentation-first protocol. Software, automation, persistent memory, integrations, and product redesign are not part of this validation.

## Test next

Run the three handoffs in the [V0 validation plan](v0-validation-plan.md). Compare what a user normally repeats with what still must be supplied after the reviewed Snapshot.

## Do not build yet

Do not build a CLI, browser extension, MCP server, app, agent workflow, or automated Snapshot generator until repeated manual tests show a stable problem that the current format cannot solve.
