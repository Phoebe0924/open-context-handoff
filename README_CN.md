# OCH — Open Context Handoff

## 别再每开一个 AI 窗口，就重新讲一遍你的故事。

OCH 是一个轻量协议，用来让你在 ChatGPT / Claude / Gemini / Cursor / Claude Code 之间，带着上下文继续工作。

---

## 🧠 问题是什么

如果你重度使用 AI，你一定经历过这种循环：

你打开一个新窗口，然后开始：

- “我们刚刚在做一个项目……”
- “这个背景是……”
- “这个方案我们已经否掉了……”
- “现在进度在这里……”

然后你会发现：

👉 你不是在继续工作  
👉 你是在重新上岗

更糟的是：

AI 会开始“自己补故事”。

有时候补对了。  
有时候补得很自信，但完全不对。

---

## 🪦 真实体验

每一个新 AI 窗口，都像：

> 你刚睡醒在一个平行宇宙，  
> 没人记得你昨天做了什么。

你只能重新证明一次：

你是谁，你在做什么，你已经走到哪里。

---

## 💡 核心洞察

这不是单纯的记忆问题。

你丢失的不是信息。

你丢失的是：

> **思考的连续性。**

更深一层说，这是：

> **上下文所有权问题。**

你无法明确决定：

- 哪些信息应该被带到下一个窗口
- 哪些决定已经定稿
- 哪些方案不该再被重新讨论
- 下一个 AI 应该从哪里继续

---

## 📦 OCH 是什么

OCH 会帮你把当前对话压缩成一份 **上下文快照（Context Snapshot）**。

你可以把它理解成：

> AI 时代的接班便条。

它不保存完整聊天记录。

它只保存足够让另一个 AI，或者三天后的你，快速重新进入状态的关键锚点。

## ⚙️ 怎么用（V0）

1. 对话快结束时生成 Context Snapshot
2. 花 5 秒确认内容有没有问题  
3. 复制到新窗口  
4. 从 NEXT ACTION 继续  

目标不是完美恢复全部上下文。

目标是：

> 让你不用每次都从零开始解释。

---

## 🧩 Context Snapshot 包含什么

```markdown
## OCH Snapshot

WHAT WE'RE DOING
[当前在做什么]

WHERE WE ARE
Done:
[已经完成什么]

Current:
[现在卡在哪里]

HARD DECISIONS
- [已经决定、不需要重开的判断]

NEXT ACTION
- [下一步具体动作，只写一条]

CONTEXT NOTES
- [少量背景信息，拿不准的不写]
```

## 🧩 核心原则

> AI 负责起草。  
> 人类负责确认。  
> 上下文属于用户，而不是平台。

---

## 🚫 OCH 不是什么

OCH 不是：

- 记忆系统
- AI Agent
- 自动化工具
- 知识库
- 聊天记录归档器

它只做一件小事：

> 让你不用每次都重新讲故事。

---

## 🧭 为什么要做这个

现在的 AI 越来越强。

但人的工作流越来越碎：

- 一个问题问 ChatGPT
- 一个角度问 Claude
- 一个验证问 Gemini
- 一个执行交给 Cursor / Claude Code

工具越来越多，思考却越来越容易断。

OCH 想解决的不是“AI 记不住我”。

而是：

> 我能不能自己决定，什么上下文应该被带到下一站？

---

## 🧪 当前状态

这是 V0。

故意很小。

不做插件。  
不做自动化。  
不做复杂系统。

先验证一件事：

> 一份简单的上下文快照，能不能减少 70% 的重复解释？

---
